In Chapter 14, you will note that in the code example for the Button class, the author has included a triple-quoted strings for the class, and functions. In fact, in most of the code provided in the book, triple-quoted strings have been provided for modules, methods, functions and classes. Discuss how these docstrings help programmers with their workflow. Remember your experience and the internet are your friends.

Docstrings are very useful in programming, especially for students. A docstring is written with triple quotes, and it explains what the class or function is doing.
 I think this makes coding much easier to understand.
 From my experience, when I go back to my code after some time, I sometimes forget what I was doing. 
The docstrings help me remember the purpose of the function or class without reading all the code again. This saves me time and makes my work faster. 
Also, docstrings are helpful when working with other people.
 If someone else reads my code, they can understand it more easily because of the explanation. Without docstrings, the code can be confusing, 
especially if the project is big. Another good thing is that some tools in Python can use docstrings to create documentation automatically. 
This is helpful because I don’t need to write separate notes. In my opinion, docstrings are very important. They help me stay organized, understand my code better, and make my workflow smoother as a student learning programming.
